# Daily AI growth loop

## Daily gate

Run exactly one cycle for the current date:

1. Read the prior post's 5h, 12h, and 24h observations without overwriting earlier rows.
2. Compare the 24h result with the rolling seven-post median when enough comparable posts exist.
3. Classify the limiting layer: distribution, packaging, content value, CTA, audience fit, or insufficient data.
4. Record direct evidence and one primary strategy change.
5. Update the audience hypothesis, winning-pattern library, and elimination register only when their evidence rules are met.
6. Generate today's content, image assets, and publishing task.
7. Complete human review and publication, then create real 5h, 12h, and 24h follow-ups.

Do not prepare another day's finished content during this cycle.

## Decision hierarchy

- Low views or exposure: test title/cover packaging; do not reject the content model.
- Exposure with weak clicks: change the target cue, conflict, specificity, or benefit on the title/cover.
- Clicks with weak saves/comments: improve the framework, steps, examples, or reusable value.
- Saves with weak private messages: change the CTA promise and keyword.
- Qualified inquiries: preserve the audience, pain point, model, and CTA; create one controlled successor only when it becomes that day's selected test.
- Missing metrics: record `insufficient_data`; do not score missing values as zero.

## Learning rules

- Promote a direction only after at least two comparable posts outperform the rolling baseline and show a positive acquisition signal.
- Place a model on watch after two interpretable underperforming tests.
- Eliminate a model only after three controlled tests fail with sufficient distribution; one low-distribution post is never enough.
- Preserve raw data and the exact strategy state that produced each post.
