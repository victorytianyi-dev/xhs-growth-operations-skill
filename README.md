# 小红书增长运营 Skill

一个面向 Codex 的“小红书图文增长运营”工作流。它把内容复盘、当天选题、图文素材包、创作中心草稿填写、人工审核，以及 5/12/24 小时数据复盘串成一条可追溯流程。

## 它能做什么

- 读取上一条内容的真实数据，判断是标题封面、内容价值、互动入口还是受众匹配的问题；
- 每天只生成当天的一篇内容，避免提前批量生产后失去数据反馈；
- 将复杂的 AI、开源项目或经营案例，改写成普通用户能快速看懂的“具体场景 + 实际动作 + 可验证结果”；
- 检查 `cover.png` 与 `page1.png` 至 `page7.png` 是否完整、顺序正确、尺寸为 1080×1440；
- 在小红书创作中心上传图片并填写标题、正文和标签，保留可恢复的进度记录；
- 在最终发布前停下，交给人工检查并点击发布；
- 在发布后 5、12、24 小时记录真实数据，沉淀有效内容方向与淘汰规则；
- 遇到上传失败、旧页面失效、文件访问权限未开启、登录或风控时，按安全恢复流程处理，不绕过平台限制。

## 核心规则

1. 标题不超过 20 个汉字，优先使用具体数字、时间、冲突或收益，但绝不编造。
2. 每页只讲一个重点，用普通职场语言，不把“智能体、工作流、RAG”等术语堆给读者。
3. 已审核的图片文字、标题、正文和标签视为锁定稿；任何修改都必须可见、可复核。
4. 公开内容以分享为主：用具体场景与做法建立信任，不默认加入报价、交付承诺或硬性营销。
5. AI 可以准备素材、上传并填写草稿；最终点击“发布”必须由人完成或在最终页面获得一条新的明确授权。
6. 不保存密码、Cookie、验证码、客户身份或浏览器会话，也不绕过登录、验证码、风控或平台限制。

## 使用方式

将 [`skill/xhs-growth-operations`](skill/xhs-growth-operations) 安装到 Codex Skills 目录，或在项目中保留该目录后让 Codex 调用：

```text
使用 $xhs-growth-operations 复盘上一条小红书内容，
只生成今天的图文发布任务；上传并填写创作中心草稿，
在最终发布前停下让我审核。
```

Skill 会在需要时读取自身的规则、数据追踪、故障恢复与记录结构说明。

## 适用范围

适用于以图文笔记为主、希望通过真实数据持续优化内容的个人博主、AI 应用分享账号和小型团队。它不是批量代发工具，也不替代内容事实核验与人工最终发布。
